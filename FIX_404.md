# Fix: 404 NOT_FOUND Error on Vercel

## What Went Wrong

The 404 error means Vercel can't find your API endpoint. This usually happens because the file structure or configuration is incorrect.

## Quick Fix

Follow these steps exactly:

### 1. Correct File Structure

Your project folder should look like this:

```
gujarati-article-publisher/
├── api/
│   └── publish.js          (the API file)
├── package.json
├── vercel.json
└── .gitignore
```

### 2. Check Your Files

Make sure you have these exact files:

**api/publish.js** - The backend code (updated version provided)
**package.json** - Dependencies list
**vercel.json** - Configuration file (updated version provided)

### 3. Update Your Code

Delete the old `api/publish-article.js` file and replace it with `api/publish.js` (note the filename change).

### 4. Push Changes to GitHub

In your terminal:

```
git add .
git commit -m "Fix API endpoint and Vercel configuration"
git push origin main
```

### 5. Redeploy on Vercel

Go to https://vercel.com → Your Project → Deployments

Click the most recent deployment and look at the logs. Or simply make a new push to GitHub and Vercel will automatically redeploy.

### 6. Test Your API

After deployment, test with this curl command (replace with your actual Vercel URL):

```
curl -X POST https://your-project.vercel.app/api/publish \
  -H "Content-Type: application/json" \
  -d '{
    "action": "generate",
    "articleTitle": "Test Article",
    "articleContent": "This is test content"
  }'
```

You should get a JSON response with Gujarati content.

## Common Issues

### Issue: Still getting 404

**Solution:** Go to Vercel dashboard → Your Project → Deployments. Click on the latest deployment and scroll down to see the "Functions" section. It should show `api/publish.js`. If it's not there or shows an error, your files aren't in the right place.

### Issue: Environment variables not working

**Solution:** In Vercel dashboard → Settings → Environment Variables, make sure you have:
- CLAUDE_API_KEY
- WORDPRESS_PASSWORD
- WORDPRESS_URL

After adding them, redeploy your project.

### Issue: Still not working

**Solution:** Check your Vercel project logs:
1. Go to https://vercel.com → Your Project
2. Click "Deployments"
3. Click the latest deployment
4. Scroll down to "Function Logs"
5. Look for error messages

Copy the error message and I can help you fix it!

## Verify Deployment

If you see "api/publish.js → 200" in the functions list, your API is working correctly!

The API endpoint is: `https://your-project.vercel.app/api/publish`

Use this URL in your web app to make requests.
